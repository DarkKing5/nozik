### 📌 Задание 1: Кластеризация K-средних (Неделя 12)
**Файл:** `week12_kmeans_clustering.py`  
**Методичка:** Лабораторная работа №12, стр. 107–113

#### Что делает:
- **Группирует клиентов** по схожести их текстовых отзывов
- Преобразует текст в числовые признаки через **TF-IDF** (Term Frequency – Inverse Document Frequency)
- Определяет оптимальное число кластеров **методом локтя** (Elbow Method)
- Визуализирует кластеры через **PCA** (метод главных компонент)

#### Алгоритм K-средних (из методички):
1. Случайным образом создаются **k точек** (центры кластеров)
2. Для каждой точки ставится **ближайший к ней центр** кластера
3. Вычисляются **средние арифметические** точек каждого кластера → новые центры
4. Шаги 2 и 3 повторяются, пока центры **не перестанут меняться**

#### Ключевой код (как в методичке):
```python
from sklearn.cluster import KMeans

kmeans = KMeans(n_clusters=4, random_state=42)
clusters = kmeans.fit_predict(X_tfidf)
```

#### Результаты:
- 4 кластера клиентов с разными паттернами отзывов
- Silhouette Score для оценки качества кластеризации
- Графики: `week12_elbow_method.png`, `week12_clusters_pca.png`, `week12_clusters_departments.png`

#### Что отвечать на защите:
> «Мы использовали алгоритм K-средних для группировки клиентов. Сначала тексты отзывов преобразовали в числовые векторы через TF-IDF. Затем методом локтя определили оптимальное число кластеров k=4. Алгоритм итеративно обновляет центры кластеров до сходимости. Визуализация через PCA показывает чёткое разделение на группы.»

### 📌 Задание 2: SVM — Метод опорных векторов (Неделя 13)
**Файл:** `week13_svm_classification.py`  
**Методичка:** Лабораторная работа №13, стр. 114–117  
**⚠️ ВАЖНО: Используется до 100 строк данных (требование задания)**

#### Что делает:
- **Классифицирует отзывы** на положительные (Rating ≥ 4) и отрицательные (Rating < 4)
- Сравнивает **4 типа ядер**: линейное, RBF, полиномиальное, сигмоидное
- Визуализирует **границы принятия решений** (decision boundaries)

#### Теория SVM (из методички):
> SVM (Support Vector Machines) – алгоритм машинного обучения, который строит **гиперплоскость** в n-мерном пространстве для разделения объектов на классы. Гиперплоскость выбирается так, чтобы **максимизировать зазор (margin)** между ней и ближайшими объектами разных классов — **опорными векторами**.

#### 4 ядра SVM:
| Ядро | Описание | Когда применять |
|------|----------|-----------------|
| **Linear** | Скалярное произведение | Линейно разделимые данные |
| **RBF** | Радиально-базисная функция | Нелинейные данные (универсальное) |
| **Polynomial** | Полиномиальная функция | Данные с полиномиальными зависимостями |
| **Sigmoid** | Сигмоидная функция | Моделирование нейронных сетей |

#### Ключевой код (как в методичке):
```python
from sklearn import svm
from sklearn.metrics import accuracy_score

C = 1.0  # параметр регуляризации SVM
linear_svc = svm.SVC(kernel='linear', C=C).fit(X_train, y_train)
rbf_svc = svm.SVC(kernel='rbf', gamma=0.7, C=C).fit(X_train, y_train)
poly_svc = svm.SVC(kernel='poly', degree=2, C=C).fit(X_train, y_train)
sig_svc = svm.SVC(kernel='sigmoid', C=C).fit(X_train, y_train)

print('Accuracy of linear kernel:', accuracy_score(y_test, linear_svc.predict(X_test)))
```

#### Что отвечать на защите:
> «SVM строит гиперплоскость для разделения классов с максимальным зазором. Мы сравнили 4 ядра на 100 строках данных. Линейное ядро строит прямую границу, RBF создаёт радиально-симметричный колокол вокруг каждого опорного вектора. Полиномиальное ядро вводит нелинейность через степень. Лучший результат показало линейное/RBF ядро.»

### 📌 Задание 3: Нейронная сеть (Недели 9–10)
**Файл:** `week9_neural_network.py`  
**Методичка:** Лабораторные работы №9–10, стр. 88–103

#### Что делает:
- Реализует **нейронную сеть с нуля** (без TensorFlow/Keras!)
- Архитектура: **N входов → 2 скрытых нейрона → 1 выход**
- Прогнозирует вероятность **положительного/отрицательного** отзыва

#### Ключевые понятия (из методички):

**Функция активации — Сигмоида:**
```
f(x) = 1 / (1 + e^(-x))
f'(x) = f(x) * (1 - f(x))
```

**Функция потерь — MSE (средняя квадратичная ошибка):**
```
MSE = (1/n) * Σ(y_true - y_pred)²
```

**Метод обратного распространения (backpropagation):**
> Метод расчёт частных производных «от конца к началу» — вычисляем dL/dw для каждого веса через цепное правило

**Стохастический градиентный спуск (SGD):**
```
w₁ ← w₁ - η * (∂L/∂w₁)
```
> где η (eta) — скорость обучения

#### Ключевой код (как в методичке):
```python
import numpy as np

def sigmoid(x):
    return 1 / (1 + np.exp(-x))

def deriv_sigmoid(x):
    fx = sigmoid(x)
    return fx * (1 - fx)

def mse_loss(y_true, y_pred):
    return ((y_true - y_pred) ** 2).mean()

# Обновление весов (SGD):
self.w1 -= learn_rate * d_L_d_ypred * d_ypred_d_h1 * d_h1_d_w1
```

#### Процесс обучения:
1. Выбираем **одно наблюдение** из набора данных → стохастический спуск
2. Считаем **все частные производные** потерь по весам и порогам
3. Используем **формулу обновления** для каждого веса и порога
4. Снова переходим к **шагу 1**

#### Что отвечать на защите:
> «Нейронная сеть реализована с нуля. Используется сигмоидная функция активации, MSE как функция потерь, и стохастический градиентный спуск для оптимизации. Обратное распространение ошибки (backpropagation) вычисляет частные производные через цепное правило — от выхода к входу. На каждом шаге веса обновляются: w ← w - η·(∂L/∂w).»

### 📌 Задание 4: K-ближайших соседей (Неделя 11)
**Файл:** `week11_knn_classifier.py`  
**Методичка:** Лабораторная работа №11, стр. 104–106

#### Что делает:
- **Прогнозирует оценку** (рейтинг 1–5) клиента по схожим отзывам
- Определяет **оптимальное k** перебором от 1 до 20
- Сравнивает **3 метрики расстояния**: евклидова, манхэттенская, косинусная

#### Принцип работы KNN (из методички):
1. Вычисляется **расстояние** между тестовым и всеми обучающими образцами
2. Выбирается **k-ближайших образцов** (соседей), где число k задаётся заранее
3. Итоговый прогноз — **мода** (самый частый класс) среди k соседей
4. Предыдущие шаги повторяются для всех тестовых образцов

#### Метрики расстояния:
| Метрика | Формула | Описание |
|---------|---------|----------|
| **Евклидова** | √Σ(aᵢ - bᵢ)² | Длина отрезка между точками |
| **Манхэттенская** | Σ\|aᵢ - bᵢ\| | Сумма модулей разностей координат |
| **Косинусная** | 1 - cos(a,b) | Угол между векторами |

#### Оптимизации (Из методички):
- **BallTree** — древовидная структура с гиперсферами для быстрого поиска соседей
- **KD-Tree** — разбиение пространства гиперплоскостями

#### Ключевой код:
```python
from sklearn.neighbors import KNeighborsClassifier

knn = KNeighborsClassifier(n_neighbors=k, metric='euclidean')
knn.fit(X_train, y_train)
y_pred = knn.predict(X_test)
```

#### Что отвечать на защите:
> «KNN — алгоритм, который классифицирует новый объект по голосованию k ближайших соседей. Расстояние вычисляется евклидовой метрикой. Оптимальное k мы подобрали перебором — максимум accuracy достигается при k=N. Для ускорения на больших данных используются BallTree и KD-Tree.»

### 📌 Задание 5: Обучение с подкреплением (Неделя 14)
**Файл:** `week14_reinforcement_learning.py`  
**Методичка:** Лабораторная работа №14, стр. 118–119

#### Что делает:
- Моделирует **среду обслуживания клиентов** на основе реальных данных рейтингов
- Обучает **Q-Learning агента** оптимальной стратегии обслуживания
- Визуализирует **Q-таблицу** и процесс обучения

#### Теория (из методички):

**Марковский процесс принятия решения (S, A, P, R):**
- **S** — множество состояний (уровни удовлетворённости 0–4)
- **A** — множество действий (стратегии обслуживания)
- **P: S×A → S** — функция переходов (следующее состояние)
- **R: S×A → ℝ** — функция награды

**Уравнение Беллмана для Q-обучения:**
```
Q(s,a) ← Q(s,a) + α[R(s,a) + γ·max_a'(Q(s',a')) - Q(s,a)]
```
> где α — скорость обучения, γ — коэффициент дисконтирования

**Epsilon-greedy стратегия:**
- С вероятностью ε — **случайное действие** (исследование среды)
- С вероятностью (1-ε) — **лучшее известное действие** (использование знаний)
- ε постепенно уменьшается: от исследования к использованию

#### Состояния и действия в нашей модели:

| Состояние | Описание | Лучшее действие (выученное) |
|-----------|----------|-----------------------------|
| 0: Очень недоволен | Rating = 1 | Приоритетная поддержка |
| 1: Недоволен | Rating = 2 | Скидка/промо |
| 2: Нейтрален | Rating = 3 | Персональная рекомендация |
| 3: Доволен | Rating = 4 | Рекомендация/Лояльность |
| 4: Очень доволен | Rating = 5 | Стандарт/Лояльность |

#### Что отвечать на защите:
> «Обучение с подкреплением — подход, когда агент учится через взаимодействие со средой. Мы смоделировали среду обслуживания клиентов с 5 состояниями и 5 действиями. Q-Learning использует уравнение Беллмана для обновления Q-таблицы. Epsilon-greedy стратегия балансирует между исследованием и использованием. Результат — оптимальная стратегия: недовольным клиентам предлагать поддержку, довольным — программу лояльности.»
