# 📌 ЗАДАНИЕ 1: Кластеризация K-средних

**Файл:** `week12_kmeans_clustering.py`

**Данные:** весь датасет (22 641 отзыв)

### Что делает скрипт — по шагам

#### Шаг 1: Загрузка данных
```python
df = pd.read_csv('Womens Clothing E-Commerce Reviews.csv', index_col=0)
df = df.dropna(subset=['Review Text'])  # убираем строки без отзыва
```
Загружаем датасет, убираем строки где нет текста отзыва (их ~5000).

#### Шаг 2: TF-IDF — текст → числа
```python
from sklearn.feature_extraction.text import TfidfVectorizer

tfidf = TfidfVectorizer(max_features=1000, stop_words='english')
X_tfidf = tfidf.fit_transform(df['Review Text'])
```
**TF-IDF** (Term Frequency – Inverse Document Frequency) превращает текст в числовой вектор.
- Чем **чаще** слово встречается в отзыве — тем выше его вес
- Чем **реже** слово встречается во всём датасете — тем оно информативнее
- `max_features=1000` — берём только 1000 самых важных слов
- `stop_words='english'` — убираем "the", "and", "is"...

Результат: матрица `(22641, 1000)` — 22641 отзыв, каждый описан 1000 числами.

#### Шаг 3: Метод локтя (как выбрать k?)
```python
for k in range(2, 8):
    kmeans = KMeans(n_clusters=k, random_state=42)
    kmeans.fit(X_tfidf)
    inertias.append(kmeans.inertia_)  # сумма расстояний до центров
```
**Инерция** = сумма квадратов расстояний от каждой точки до своего центра.
При увеличении k инерция падает, но если падает медленно — смысла добавлять кластеры нет.
Смотрим на "локоть" — место где кривая перестаёт резко снижаться. У нас: **k=4**.

График: `week12_elbow_method.png`

#### Шаг 4: Кластеризация
```python
kmeans = KMeans(n_clusters=4, random_state=42, n_init=10)
clusters = kmeans.fit_predict(X_tfidf)
df['Cluster'] = clusters
```
Алгоритм K-средних (из методички стр. 108–109):
1. Случайно создаём k точек — начальные центры
2. Каждой точке назначаем ближайший центр
3. Пересчитываем центры как среднее арифметическое их точек
4. Повторяем 2–3 пока центры не перестанут меняться

# 📌 ЗАДАНИЕ 3: Нейронная сеть

**Файл:** `week9_neural_network.py`  
**Методичка:** Лабораторные работы №9–10, стр. 88–103  
**Нейросеть написана ВРУЧНУЮ — без TensorFlow/Keras!**

---

### Что делает скрипт — по шагам

#### Шаг 1: Сигмоидная функция активации
```python
def sigmoid(x):
    # f(x) = 1 / (1 + e^(-x))
    # Сжимает любое число в диапазон [0, 1]
    return 1 / (1 + np.exp(-np.clip(x, -500, 500)))

def deriv_sigmoid(x):
    # Производная: f'(x) = f(x) * (1 - f(x))
    # Нужна для обратного распространения
    fx = sigmoid(x)
    return fx * (1 - fx)
```

#### Шаг 2: Функция потерь MSE
```python
def mse_loss(y_true, y_pred):
    # MSE = (1/n) * Σ(y_true - y_pred)²
    # Чем меньше — тем лучше модель
    return ((y_true - y_pred) ** 2).mean()
```

#### Шаг 3: Архитектура нейронной сети

```
Входы (3 нейрона)       Скрытый слой (2 нейрона)     Выход (1 нейрон)
   Age          ──┐
   Recommended  ──┼──→  h1 (sigmoid) ──┐
   Feedback     ──┘                    ├──→  o1 (sigmoid) → 0 или 1
                  └──→  h2 (sigmoid) ──┘
```

Входные признаки (без Rating! — это утечка данных):
- `Age` — возраст клиента
- `Recommended IND` — рекомендует ли товар (0/1)
- `Positive Feedback Count` — количество лайков отзыву

#### Шаг 4: Прямой проход (feedforward)
```python
def feedforward(self, x):
    # Скрытый нейрон h1
    h1 = sigmoid(np.dot(x, self.w_hidden1) + self.b1)
    # Скрытый нейрон h2
    h2 = sigmoid(np.dot(x, self.w_hidden2) + self.b2)
    # Выходной нейрон
    o1 = sigmoid(self.w_output[0]*h1 + self.w_output[1]*h2 + self.b3)
    return o1  # вероятность 0..1
```

#### Шаг 5: Обратное распространение (backpropagation)
```python
# Вычисляем частные производные "от конца к началу"
d_L_d_ypred = -2 * (y_true - y_pred)          # dL/d(o1)

# Нейрон выхода o1:
d_ypred_d_w5 = h1 * deriv_sigmoid(sum_o1)      # d(o1)/dw5
d_ypred_d_h1 = self.w_output[0] * deriv_sigmoid(sum_o1)

# Нейрон h1:
d_h1_d_w = x * deriv_sigmoid(sum_h1)          # d(h1)/dw1

# Обновляем вес по SGD:
# w ← w - η * (∂L/∂w)
self.w_hidden1 -= learn_rate * d_L_d_ypred * d_ypred_d_h1 * d_h1_d_w
```

Это **цепное правило** дифференцирования — умножаем производные шаг за шагом от выхода к входу.

#### Шаг 6: Обучение
```python
network = OurNeuralNetwork(n_features=3)
losses = network.train(X_train, y_train, epochs=300, learn_rate=0.05)

# Эпоха 0:   loss = 0.1107  (плохо)
# Эпоха 50:  loss = 0.0583  (учится!)
# Эпоха 250: loss = 0.0583  (стабилизировалась)
```

#### Результат
```

- Отрицательных: precision=0.97, recall=0.72
- Положительных: precision=0.91, recall=0.99
```

### 🗣️ Что говорить на защите

> «Нейронная сеть реализована с нуля, без готовых фреймворков. Архитектура: 3 входа → 2 скрытых нейрона → 1 выход. Функция активации — сигмоида, которая сжимает значение в [0,1]. Функция потерь — MSE. Оптимизация — стохастический градиентный спуск: на каждом шаге обучения методом обратного распространения вычисляем частные производные потерь по каждому весу через цепное правило, затем обновляем веса: w ← w − η·(∂L/∂w). # 📌 ЗАДАНИЕ 4: K-ближайших соседей

**Файл:** `week11_knn_classifier.py`  
**Методичка:** Лабораторная работа №11, стр. 104–106  
**Задача:** предсказать рейтинг (1–5) по тексту отзыва и другим признакам

---

### Что делает скрипт — по шагам

#### Шаг 1: Признаки
```python
# Числовые признаки
X_num = df[['Age', 'Recommended IND', 'Positive Feedback Count']].fillna(0).values
```

#### Шаг 2: Принцип KNN (4 шага из методички)
1. Вычислить **Евклидово расстояние** от тестового образца до всех обучающих:
   `d = √Σ(aᵢ − bᵢ)²`
2. Выбрать **k ближайших** по расстоянию
3. Определить итоговый класс как **моду** (самый частый класс среди k соседей)
4. Повторить для всех тестовых образцов

#### Шаг 3: Выбор оптимального k
```python
for k in range(1, 21):

    acc = accuracy_score(y_test, knn.predict(X_test))

```

#### Шаг 4: Сравнение метрик расстояния

```python
# Евклидова:     √Σ(aᵢ-bᵢ)²     → 57.74%
```

# 📌 ЗАДАНИЕ 5: Обучение с подкреплением

**Файл:** `week14_reinforcement_learning.py`  
**Методичка:** Лабораторная работа №14, стр. 118–119  
**Задача:** агент учится оптимизировать стратегию обслуживания клиентов

---

### Что делает скрипт — по шагам

#### Шаг 1: Формализация задачи

**Марковский процесс принятия решения (S, A, P, R):**

| Компонент | Описание | В нашей задаче |
|-----------|----------|----------------|
| **S** — состояния | Ситуации, в которых может быть агент | 5 уровней удовлетворённости (1–5) |
| **A** — действия | Что может делать агент | 5 стратегий обслуживания |
| **P: S×A→S** | Функция переходов | Как меняется довольность после действия |
| **R: S×A→ℝ** | Функция награды | +2 улучшилось, -2 ухудшилось |

```python
# 5 состояний (со 0 = Rating 1 до 4 = Rating 5)
states = ['Очень недоволен', 'Недоволен', 'Нейтрален', 'Доволен', 'Очень доволен']

# 5 действий
actions = ['Стандарт', 'Рекомендация', 'Скидка', 'Поддержка', 'Лояльность']

# Матрица наград R[state][action]
R = [
    [-2, -1,  1,  2,  0],  # Очень недоволен: поддержка → +2
    [-1,  0,  2,  1,  0],  # Недоволен: скидка → +2
    [ 0,  1,  1,  0,  1],  # Нейтрален: рекомендация → +1
    [ 1,  2,  0,  0,  2],  # Доволен: рекомендация/лояльность → +2
    [ 2,  1, -1, -1,  2],  # Очень доволен: стандарт/лояльность → +2
]
```

#### Шаг 2: Q-таблица
```python
# Q[состояние][действие] = "ожидаемая накопленная награда"
# Инициализируем нулями
Q = np.zeros((n_states, n_actions))
```

Q-таблица — это "мозг" агента. Она содержит оценку того, насколько выгодно от конкретного estado сделать конкретное действие.

#### Шаг 3: Q-Learning (уравнение Беллмана)
```python
# Для каждого эпизода, для каждого шага:
Q[s][a] = Q[s][a] + alpha * (
    reward                    # полученная награда прямо сейчас
    + gamma * np.max(Q[s_next])  # максимальная будущая награда
    - Q[s][a]                 # текущая оценка (корректируем)
)
```

- `alpha = 0.1` — **скорость обучения** (насколько быстро обновляем Q)
- `gamma = 0.9` — **дисконт** (будущие награды чуть менее ценны, чем текущие)

#### Шаг 4: Epsilon-greedy стратегия
```python
epsilon = 1.0  # начальная вероятность случайного действия
epsilon_decay = 0.995  # ε уменьшается с каждым эпизодом
epsilon_min = 0.01

if random() < epsilon:
    action = random_action()   # ИССЛЕДОВАНИЕ: пробуем новое
else:
    action = argmax(Q[state])  # ИСПОЛЬЗОВАНИЕ: лучшее известное
```

Сначала уделяем: **много исследования** → постепенно → **больше использования**.

#### Шаг 5: 5000 эпизодов обучения

```
Эпизод 1000/5000: Средняя награда = 37.89, ε = 0.01
Эпизод 5000/5000: Средняя награда = 37.54, ε = 0.01
```

#### Шаг 6: Выученная стратегия

```
Очень недоволен → Приоритетная ПОДДЕРЖКА  (Q=18.49)
Недоволен       → СКИДКА / промо          (Q=18.29)
Нейтрален       → Персональная РЕКОМЕНДАЦИЯ (Q=18.12)
Доволен         → РЕКОМЕНДАЦИЯ + Лояльность (Q=19.03)
Очень доволен   → Стандарт (не мешать)    (Q=19.00)
```

Это **бизнес-логика**: недовольным — скидку и поддержку, довольным — лояльность и персонализацию.
